## 1. 为什么需要 Transformer？

### 1.1 RNN 的瓶颈

在 Transformer 之前，序列建模的主力是 RNN/LSTM：

```
RNN 的处理方式（串行）：
x₁ → h₁ → x₂ → h₂ → x₃ → h₃ → ... → xₙ → hₙ
      ↑           ↑           ↑
      信息必须一步步传递，不能跳步
```

**三个致命问题：**

| 问题                    | 表现                     | 后果                                          |
| ----------------------- | ------------------------ | --------------------------------------------- |
| **串行计算**      | 必须按顺序处理，不能并行 | 训练慢，GPU 利用率低                          |
| **长距离依赖**    | 信息经过多次传递后衰减   | "主语在第 1 个词，谓语在第 50 个词" → 学不到 |
| **梯度消失/爆炸** | 反向传播梯度连乘         | 深层网络训练困难                              |

### 1.2 Attention 的突破

2014 年，Bahdanau 等人提出了 Attention 机制：**让模型直接"看到"序列中的任意位置，不需要一步步传递**。

```
Attention 的方式（并行）：
x₁ ←——→ x₂ ←——→ x₃ ←——→ ... ←——→ xₙ
  ↘    ↗   ↘    ↗
   任意两个位置都可以直接交互
```

Transformer 的核心贡献：**完全抛弃 RNN，只用 Attention 就能建模序列**。

## 2. Self-Attention：注意力是怎么工作的？

### 2.1 直觉：图书馆类比

想象你在图书馆查资料：

| 概念                | 类比                         | 作用             |
| ------------------- | ---------------------------- | ---------------- |
| **Query (Q)** | 你的问题："什么是量子计算？" | 你在寻找什么     |
| **Key (K)**   | 每本书的标题/摘要            | 每本书能提供什么 |
| **Value (V)** | 每本书的实际内容             | 每本书的具体信息 |

过程：

1. 用你的 **Query** 和每本书的 **Key** 比较相关性 → 得到注意力权重
2. 根据权重对 **Value** 加权求和 → 得到最终答案

```
"量子计算" 这个 Query
    和 "物理学导论" 的 Key → 相关性 0.6
    和 "烹饪大全"   的 Key → 相关性 0.01
    和 "量子力学"   的 Key → 相关性 0.9

最终答案 = 0.6 × 物理学内容 + 0.01 × 烹饪内容 + 0.9 × 量子力学内容
         ≈ 主要来自量子力学那本书
```

### 2.2 数学公式

对于输入序列 X（每个词是一个向量），Self-Attention 的计算：

```
步骤 1: 线性变换得到 Q, K, V
    Q = X · W_Q    (Query 矩阵)
    K = X · W_K    (Key 矩阵)
    V = X · W_V    (Value 矩阵)

步骤 2: 计算注意力分数
    scores = Q · K^T / √d_k

步骤 3: Softmax 归一化
    weights = softmax(scores)

步骤 4: 加权求和
    output = weights · V
```

### 2.3 为什么要除以 √d_k？

假设 Q 和 K 的每个元素是均值为 0、方差为 1 的独立随机变量。

Q·K^T 的每个元素是 d_k 个乘积之和，方差 = d_k。

当 d_k 很大时（比如 64），点积值会很大 → softmax 的输入值很大 → softmax 输出接近 one-hot → **梯度接近 0，学不动了**。

除以 √d_k 把方差压回 1，让 softmax 的输入在合理范围内。

```
d_k = 64 时：
  不除：点积值可能达到 ±30 → softmax ≈ [0, 0, 1, 0, ...] → 梯度消失
  除以 8：点积值在 ±3 左右 → softmax 分布平滑 → 梯度正常
```

## 3. Multi-Head Attention：为什么要多头？

### 3.1 单头的局限

一个 Attention Head 只能学到**一种关系模式**。

但语言中同时存在多种关系：

- 语法关系：主语 ↔ 谓语
- 语义关系：同义词 ↔ 同义词
- 指代关系：代词 ↔ 先行词
- 位置关系：相邻词之间的关系

### 3.2 多头并行

Multi-Head Attention 把 Q, K, V 分成 h 份，每份独立做 Attention，最后拼接：

```
MultiHead(Q, K, V) = Concat(head₁, head₂, ..., headₕ) · W_O

其中 headᵢ = Attention(Q · W_Qᵢ, K · W_Kᵢ, V · W_Vᵢ)
```

论文中 h = 8，d_model = 512，所以每个 head 的维度 = 512/8 = 64。

**不同 head 学到的不同模式（可视化研究发现）：**

- Head 1：关注相邻词（局部语法）
- Head 2：关注动词和宾语（语义关系）
- Head 3：关注标点和句尾（句子结构）
- ...

### 3.3 直觉理解

```
单头：一个专家看全文，只能从一个角度分析
多头：8 个专家同时看全文，每人关注不同方面，最后汇总

专家1（语法）："The cat sat on the mat"
                cat ←→ sat  （主谓关系）

专家2（语义）："The cat sat on the mat"
                cat ←→ mat  （动物在哪）

专家3（位置）："The cat sat on the mat"
                The ←→ cat  （相邻关系）
```

## 4. Positional Encoding：如何告诉模型词序？

### 4.1 问题：Attention 没有位置概念

Self-Attention 是**集合操作**——打乱输入顺序，输出结果完全一样（只是顺序变了）。

但 "狗咬人" 和 "人咬狗" 意思完全不同！模型必须知道词的位置。

### 4.2 正弦/余弦位置编码

论文使用固定的正弦余弦函数：

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

其中 pos 是位置，i 是维度索引。

### 4.3 为什么用正弦余弦？

**关键性质**：PE(pos+k) 可以表示为 PE(pos) 的线性变换。

这意味着模型可以通过学习来理解**相对位置关系**（"左边第 3 个词"、"右边第 5 个词"），而不需要记住绝对位置。

### 4.4 可视化

```
位置 0:  sin(0/10000^0)  cos(0/10000^0)  sin(0/10000^2/512) ...
位置 1:  sin(1/10000^0)  cos(1/10000^0)  sin(1/10000^2/512) ...
位置 2:  sin(2/10000^0)  cos(2/10000^0)  sin(2/10000^2/512) ...
...

低维度（i 小）：周期短，变化快 → 区分相邻位置
高维度（i 大）：周期长，变化慢 → 区分远距离位置
```

最终输入 = 词嵌入 + 位置编码

## 5. Encoder 架构详解

### 5.1 单层 Encoder

```
输入 x
  │
  ├──→ Multi-Head Self-Attention ──→ Dropout
  │              │
  │     Residual Connection: x + attention_output
  │              │
  │        Layer Normalization
  │              │
  ├──→ Feed-Forward Network (FFN)
  │              │
  │        Dropout
  │              │
  │     Residual Connection: prev + ffn_output
  │              │
  │        Layer Normalization
  │              │
  └──→ 输出
```

### 5.2 Feed-Forward Network (FFN)

每个位置独立的两层全连接网络：

```
FFN(x) = max(0, x · W₁ + b₁) · W₂ + b₂

维度变化：512 → 2048 → 512
```

**为什么内层维度是 2048（4 倍 d_model）？**

- 相当于给模型一个"思考空间"：先升维提取更丰富的特征，再降维回去
- 类似 CNN 中的"瓶颈层"

### 5.3 Add & Norm（残差 + 层归一化）

```
LayerNorm(x + Sublayer(x))
```

- **残差连接**：让梯度直接跳过子层，解决深层网络的梯度消失
- **Layer Normalization**：对每个样本的特征维度做归一化（不是对 batch），稳定训练

### 5.4 完整 Encoder = 6 层堆叠

```
输入嵌入 + 位置编码
    ↓
[Encoder Layer 1]  ← Multi-Head Self-Attention + FFN
    ↓
[Encoder Layer 2]
    ↓
  ...
    ↓
[Encoder Layer 6]
    ↓
Encoder 输出（上下文表示）
```

## 6. Decoder 架构详解

### 6.1 单层 Decoder（比 Encoder 多一个子层）

```
输入 y（已生成的输出序列）
  │
  ├──→ Masked Multi-Head Self-Attention ──→ Add & Norm
  │              │
  ├──→ Multi-Head Cross-Attention ──→ Add & Norm    ← 这里用 Encoder 的输出
  │              │
  ├──→ Feed-Forward Network ──→ Add & Norm
  │
  └──→ 输出
```

### 6.2 两种特殊的 Attention

| 类型                                      | Q 来源  | K/V 来源     | 作用                       |
| ----------------------------------------- | ------- | ------------ | -------------------------- |
| **Self-Attention** (Encoder)        | 自己    | 自己         | 理解输入序列的内部关系     |
| **Masked Self-Attention** (Decoder) | 自己    | 自己         | 只看已生成的词，不看未来   |
| **Cross-Attention** (Decoder)       | Decoder | Encoder 输出 | 让 Decoder "参考" 输入序列 |

### 6.3 Causal Mask（因果掩码）

Decoder 在生成第 t 个词时，只能看到前 t-1 个词：

```
mask = [[0, -inf, -inf, -inf],
        [0,   0, -inf, -inf],
        [0,   0,   0, -inf],
        [0,   0,   0,   0]]

scores = Q·K^T + mask   ← 加上 -inf 后，softmax 输出为 0
```

这就是为什么 GPT 系列模型生成文本时是**逐词生成**的——它在架构上就看不到未来的词。

## 7. 完整的 Encoder-Decoder 流程

以机器翻译（英文→中文）为例：

```
输入: "I love you"
        ↓
[Tokenize] → [词嵌入] + [位置编码]
        ↓
[Encoder × 6层]
        ↓
Encoder 输出: 每个输入词的上下文表示
        ↓                          ↓
[Decoder × 6层] ←── Cross-Attention ──┘
  ↑
  │
  已生成的输出: "<start> 我 爱"
  ↓
[Linear + Softmax] → 下一个词的概率分布
        ↓
  预测: "你"
  ↓
  继续生成直到 <end>
```

## 9. Layer Normalization vs Batch Normalization

| 对比                 | Batch Norm            | Layer Norm         |
| -------------------- | --------------------- | ------------------ |
| 归一化维度           | 对同一特征的 batch 维 | 对同一样本的特征维 |
| 依赖 batch size      | 是                    | 否                 |
| 适合 RNN/Transformer | 差（变长序列）        | 好                 |
| 推理时行为           | 需要 running mean     | 直接计算           |

Transformer 选择 Layer Norm 的原因：

- 序列长度不固定，Batch Norm 不稳定
- Layer Norm 对每个样本独立归一化，不受 batch 中其他样本影响

## 10. 论文关键结果

### 10.1 机器翻译（WMT 2014）

| 模型                         | BLEU (EN→DE)  | BLEU (EN→FR)  | 训练时间                   |
| ---------------------------- | -------------- | -------------- | -------------------------- |
| LSTM + Attention             | 26.4           | 40.5           | 数天                       |
| **Transformer (base)** | **28.4** | **41.0** | **12 小时（8 GPU）** |
| **Transformer (big)**  | **29.3** | **41.8** | **3.5 天（8 GPU）**  |

**关键发现**：Transformer 不仅效果更好，训练速度也快了一个数量级。

### 10.2 为什么这么快？

```
RNN:  64 个时间步必须串行 → 64 次顺序计算
      每次计算涉及 hidden_size × hidden_size 的矩阵乘法

Transformer: 64 个位置全部并行 → 1 次矩阵乘法（大矩阵）
             GPU 的并行计算能力完全释放
```

## 11. 后续发展：Transformer 家族

### 11.1 三大变体

| 变体                      | 用了什么     | 代表模型        | 擅长什么               |
| ------------------------- | ------------ | --------------- | ---------------------- |
| **Encoder-only**    | 只用 Encoder | BERT, RoBERTa   | 文本理解（分类、NER）  |
| **Decoder-only**    | 只用 Decoder | GPT 系列, LLaMA | 文本生成（对话、续写） |
| **Encoder-Decoder** | 完整架构     | T5, BART, mBART | 翻译、摘要             |

### 11.2 里程碑

```
2017  Transformer 原论文（本章内容）
2018  GPT-1（Decoder-only，生成式预训练）
2018  BERT（Encoder-only，双向预训练，刷新 11 项 NLP 任务）
2020  GPT-3（1750 亿参数，Few-shot 学习）
2022  ChatGPT（InstructGPT + RLHF）
2023  GPT-4（多模态）
2024  LLaMA 3, Qwen2, DeepSeek-V2 等开源大模型
```

### 11.3 Vision Transformer (ViT)

2020 年，Google 把 Transformer 用到了图像上：

- 把图像切成 16×16 的 patch，每个 patch 当作一个"词"
- 直接用标准 Transformer Encoder 处理
- 在 ImageNet 上超越了 CNN

**意义**：证明了 Transformer 不仅能处理序列，还能处理空间数据——它是一个通用架构。

## 12. 关键术语速查

| 术语       | 英文                 | 含义                                       |
| ---------- | -------------------- | ------------------------------------------ |
| 自注意力   | Self-Attention       | 序列内部的注意力，每个位置关注其他所有位置 |
| 多头注意力 | Multi-Head Attention | 多个注意力头并行，学习不同模式             |
| 位置编码   | Positional Encoding  | 注入序列位置信息的向量                     |
| 前馈网络   | Feed-Forward Network | 每个位置独立的两层全连接网络               |
| 残差连接   | Residual Connection  | 跳跃连接，x + f(x)，缓解梯度消失           |
| 层归一化   | Layer Normalization  | 对特征维度归一化，稳定训练                 |
| 因果掩码   | Causal Mask          | 阻止 Decoder 看到未来词的掩码              |
