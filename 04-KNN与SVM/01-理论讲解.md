# KNN 与 SVM (K-Nearest Neighbors & Support Vector Machine)

## 第一部分：KNN (K-Nearest Neighbors)

### 1.1 什么是KNN？

KNN是最简单的机器学习算法之一：**找到k个最近的邻居，用他们的多数类别来分类你**。

### 1.2 直观理解

```
新样本来了！
    ↓
找到离它最近的k个训练样本
    ↓
看这k个样本的类别
    ↓
投票：哪个类别最多，就是新样本的类别
```

### 1.3 工作流程

1. **计算距离**：新样本与所有训练样本的距离
2. **找近邻**：找k个距离最小的样本
3. **投票**：这k个样本中，哪个类别最多？
4. **预测**：新样本属于多数类别

## 2. 距离度量

### 2.1 欧氏距离 (Euclidean Distance)

```
d(x, x') = √(Σ(xᵢ - x'ᵢ)²)
```

最常用。

### 2.2 曼哈顿距离 (Manhattan Distance)

```
d(x, x') = Σ|xᵢ - x'ᵢ|
```

计算快，对离群值敏感。

### 2.3 闵可夫斯基距离 (Minkowski Distance)

```
d(x, x') = (Σ|xᵢ - x'ᵢ|^p)^(1/p)
```

- p=1：曼哈顿距离
- p=2：欧氏距离

## 3. 如何选择K值？

- **K太小**（如K=1）：
  - 容易受噪声影响（过拟合）
  - 决策边界复杂

- **K太大**（如K=n）：
  - 只是预测最频繁的类（欠拟合）
  - 计算时间长

- **推荐**：√n 到 10倍于√n

## 4. KNN的优缺点

### 优点
- ✅ 算法简单，易于理解
- ✅ 无需训练
- ✅ 支持多分类
- ✅ 适应非线性边界

### 缺点
- ❌ 预测慢（需计算所有距离）
- ❌ 对特征尺度敏感（需归一化）
- ❌ 高维数据表现差（维数诅咒）
- ❌ 对不平衡数据敏感
- ❌ 内存占用大

## 5. 维数诅咒 (Curse of Dimensionality)

在高维空间中，距离变得几乎无意义。为什么？

- 所有点都离彼此很远
- 距离无法很好地反映相似度
- K个最近的点可能其实并不近

**解决方案**：
- 特征选择
- 特征提取（PCA）
- 降维

## 第二部分：SVM (Support Vector Machine)

### 1.1 什么是SVM？

SVM是一种强大的分类算法，目标是找到一条**最优的分割线**，使得：
- 两个类别分离最大
- 离分割线最近的点距离最远

### 1.2 直观理解

```
❌ ❌        ← 类别1
           _____ (分割线)
        ○ ○ ○  ← 类别2
```

SVM的目标：找到宽度最大的"走廊"

### 1.3 核心概念

**支持向量 (Support Vectors)**：离分割线最近的几个点
- 这些点完全决定了最优分割线
- 移除其他点不影响结果

**间隔 (Margin)**：分割线到最近点的距离
- SVM最大化间隔

## 2. 线性 vs 非线性

### 2.1 线性可分

```
❌ --------- ○○
   (一条直线就能分开)
```

**参数C**：误分类的惩罚
- C大：尽量不出错（可能过拟合）
- C小：允许某些错误（可能欠拟合）

### 2.2 非线性不可分

```
❌  ○  ❌
  ○   ○  
❌  ○  ❌
(直线无法分开)
```

**解决方案**：核技巧 (Kernel Trick)

## 3. 核函数 (Kernel)

把非线性问题转化为线性问题，无需显式计算高维特征。

### 3.1 常用核函数

**Linear（线性核）**
```
K(x, x') = x · x'
```
线性可分的问题。

**RBF（高斯核）**
```
K(x, x') = exp(-γ||x - x'||²)
```
最常用，适应大多数问题。

**Polynomial（多项式核）**
```
K(x, x') = (x · x' + c)^d
```
捕获高阶特征交互。

### 3.2 如何选择核？

```
线性可分？
  ↓ 是        否
Linear     RBF或Poly
```

## 4. 超参数

### 4.1 C（惩罚参数）

```
C = 1 / λ
```

- C大：正则化弱，容易过拟合
- C小：正则化强，容易欠拟合

### 4.2 γ (gamma)

RBF核的参数

- γ大：决策边界复杂，易过拟合
- γ小：决策边界平滑，易欠拟合

## 5. SVM的优缺点

### 优点
- ✅ 高维数据表现好
- ✅ 内存高效
- ✅ 核函数灵活
- ✅ 有坚实的理论基础

### 缺点
- ❌ 对特征缩放敏感
- ❌ 模型不易解释
- ❌ 大数据集速度慢
- ❌ 超参数调优复杂

## 6. KNN vs SVM vs 逻辑回归

| 特点 | KNN | SVM | 逻辑回归 |
|------|-----|-----|---------|
| 速度（预测） | 慢 | 中等 | 快 |
| 速度（训练） | 快（无训练） | 慢 | 快 |
| 非线性 | 好 | 好（核函数） | 差 |
| 可解释性 | 强 | 弱 | 强 |
| 内存占用 | 大 | 小 | 小 |
| 高维数据 | 差 | 好 | 好 |

## 7. 完整工作流程

1. **数据准备**
   - 加载数据
   - 处理缺失值

2. **特征工程**
   - **特别重要**：特征缩放（StandardScaler）
   - 特征选择

3. **划分数据集**
   - 训练集、验证集、测试集

4. **模型选择和参数调优**
   - KNN：选择K值
   - SVM：选择核函数和C、γ

5. **训练模型**

6. **评估模型**
   - 准确率、精确率、召回率
   - 混淆矩阵、ROC曲线

7. **超参数调优**
   - GridSearchCV
   - RandomizedSearchCV
