# 逻辑回归 (Logistic Regression)

## 1. 什么是逻辑回归？

逻辑回归是一种**分类**算法，尽管名字中有"回归"二字。它用于解决**二分类**问题（是/否、真/假、0/1）。

### 1.1 直观理解

想象你要判断一封邮件是否是垃圾邮件：

- **输入**：邮件的特征（发件人、内容长度、包含的关键词等）
- **输出**：垃圾邮件的概率（0到1之间）

如果概率 > 0.5，我们判断为垃圾邮件；否则判断为正常邮件。

### 1.2 为什么不能用线性回归？

线性回归输出的是连续值，可以是任意实数，不适合概率预测（必须在0-1之间）。

## 2. 数学原理

### 2.1 Sigmoid函数

为了将线性回归的输出转换为概率，我们使用**Sigmoid函数**：

```
σ(z) = 1 / (1 + e^(-z))
```

**性质**：

- σ(0) = 0.5
- σ(z) → 1 当 z → +∞
- σ(z) → 0 当 z → -∞
- 输出范围：[0, 1]

### 2.2 假设函数

```
h(x) = σ(w^T x + b) = 1 / (1 + e^(-(w^T x + b)))
```

**解释：**

- h(x) 表示样本x属于类别1的概率：P(y=1|x)
- 1 - h(x) 表示属于类别0的概率：P(y=0|x)

### 2.3 决策边界

**分类规则：**

```
if h(x) ≥ 0.5  →  预测为类别1
if h(x) < 0.5   →  预测为类别0
```

等价于：

```
if w^T x + b ≥ 0  →  类别1
if w^T x + b < 0   →  类别0
```

**决策边界**就是 w^T x + b = 0 这条线（或超平面）

### 2.4 损失函数（交叉熵）

不能用MSE，因为会导致非凸函数。使用**交叉熵损失**：


为什么逻辑回归不用MSE？

原因1：非凸性
  → 梯度下降可能陷入局部最优
  → 无法保证找到最好的解

原因2：学习速度
  → MSE的梯度在需要最大的时候反而最小
  → 优化效率低

解决方案：交叉熵
  → 凸函数，保证全局最优
  → 梯度与错误成正比，学习快速高效
  → 自然适应概率输出[0,1]


单个样本的损失：

```
L(h(x), y) = -y·log(h(x)) - (1-y)·log(1-h(x))
```

**理解：**

- 当 y=1 时，L = -log(h(x))，希望 h(x) → 1
- 当 y=0 时，L = -log(1-h(x))，希望 h(x) → 0

整体损失函数：

```
J(w,b) = -1/m Σᵢ₌₁ᵐ [yⁱ·log(h(xⁱ)) + (1-yⁱ)·log(1-h(xⁱ))]
```

### 2.5 梯度下降

**梯度计算：**

```
∂J/∂w = 1/m Σᵢ₌₁ᵐ (h(xⁱ) - yⁱ) · xⁱ
∂J/∂b = 1/m Σᵢ₌₁ᵐ (h(xⁱ) - yⁱ)
```

注意：形式上和线性回归一样，但h(x)定义不同！

**更新规则：**

```
w := w - α ∂J/∂w
b := b - α ∂J/∂b
```

## 3. 多分类问题

### 3.1 One-vs-Rest (OvR)

对K个类别，训练K个二分类器：

- 分类器1：类别1 vs 其他所有
- 分类器2：类别2 vs 其他所有
- ...
- 分类器K：类别K vs 其他所有

**预测时**：选择输出概率最高的分类器

### 3.2 Softmax回归（多项逻辑回归）

直接建立K分类模型：

```
P(y=k|x) = e^(w_k^T x) / Σⱼ₌₁ᴷ e^(w_j^T x)
```

## 4. 评估指标

### 4.1 混淆矩阵

|                    | 预测为正    | 预测为负    |
| ------------------ | ----------- | ----------- |
| **实际为正** | TP (真正例) | FN (假负例) |
| **实际为负** | FP (假正例) | TN (真负例) |

### 4.2 常用指标

**准确率 (Accuracy)**：

```
Accuracy = (TP + TN) / (TP + TN + FP + FN)
```

**精确率 (Precision)**：

```
Precision = TP / (TP + FP)
```

含义：预测为正的样本中，真正为正的比例

**召回率 (Recall / Sensitivity)**：

```
Recall = TP / (TP + FN)
```

含义：实际为正的样本中，被正确预测的比例

**F1分数**：

```
F1 = 2 · (Precision · Recall) / (Precision + Recall)
```

精确率和召回率的调和平均

**特异度 (Specificity)**：

```
Specificity = TN / (TN + FP)
```

### 4.3 ROC曲线与AUC

**ROC曲线**：

- X轴：假正例率 (FPR) = FP / (FP + TN)
- Y轴：真正例率 (TPR) = TP / (TP + FN) = Recall

**AUC** (Area Under Curve)：

- AUC = 1：完美分类器
- AUC = 0.5：随机猜测
- AUC > 0.8：通常认为不错

### 4.4 精确率-召回率曲线

当正负样本不平衡时，PR曲线比ROC更有信息量。

## 5. 正则化

### 5.1 L2正则化

```
J(w) = -1/m Σᵢ₌₁ᵐ [yⁱ·log(h(xⁱ)) + (1-yⁱ)·log(1-h(xⁱ))] + λ/(2m) Σⱼ₌₁ⁿ wⱼ²
```

### 5.2 L1正则化

```
J(w) = -1/m Σᵢ₌₁ᵐ [yⁱ·log(h(xⁱ)) + (1-yⁱ)·log(1-h(xⁱ))] + λ/m Σⱼ₌₁ⁿ |wⱼ|
```

## 6. 与线性回归对比

| 特性     | 线性回归    | 逻辑回归        |
| -------- | ----------- | --------------- |
| 激活函数 | 无          | Sigmoid         |
| 损失函数 | MSE         | 交叉熵          |
| 输出     | 实数        | 概率            |
| 用途     | 回归        | 分类            |
| 梯度形式 | (h(x)-y)·x | (σ(w^Tx)-y)·x |

## 7. 优缺点

### 优点

- 实现简单，计算效率高
- 输出有概率解释
- 对线性可分问题效果好
- 可解释性强

### 缺点

- 只能处理线性可分或近似线性可分的问题
- 对多重共线性敏感
- 对异常值敏感
- 特征工程很重要

## 8. 实际应用技巧

### 8.1 阈值调整

根据业务需求调整决策阈值：

- 严格的欺诈检测：降低阈值（提高召回率）
- 精准的营销：提高阈值（提高精确率）

### 8.2 处理不平衡数据

- 重采样（过采样/欠采样）
- SMOTE合成样本
- 调整类别权重
- 使用更合适的评估指标（F1, AUC）

### 8.3 特征工程

- 特征缩放很重要
- 可以添加多项式特征
- 可以添加交叉特征
- 去除高度相关特征
