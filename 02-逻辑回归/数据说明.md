# 逻辑回归示例 - 数据详细说明

## 📊 示例1：Iris花卉分类（二分类）

### 数据来源
- **来源**：sklearn内置数据集（不用下载）
- **加载方式**：`from sklearn.datasets import load_iris`

### 数据信息
| 属性 | 值 |
|------|-----|
| **样本总数** | 150朵花 |
| **特征数** | 4个 |
| **品种数** | 3个（但代码只用前2个做二分类） |
| **训练集** | 100样本（80%） |
| **测试集** | 20样本（20%） |

### 特征详解（4个特征都是花的**真实物理测量值**）

```
特征1: sepal length (cm)      → 花萼的长度（厘米）
特征2: sepal width (cm)       → 花萼的宽度（厘米）
特征3: petal length (cm)      → 花瓣的长度（厘米）
特征4: petal width (cm)       → 花瓣的宽度（厘米）
```

### 花的品种（3种）
```
品种0: Setosa (山鸢尾)        → 50朵
品种1: Versicolor (变色鸢尾)  → 50朵
品种2: Virginica (弗吉尼亚鸢尾) → 50朵
```

### 任务
**预测**：给定一朵花的4个测量值 → 判断它是哪个品种
```
输入: [花萼长, 花萼宽, 花瓣长, 花瓣宽]
输出: 品种类别 (Setosa/Versicolor/Virginica)
```

### 数据示例
```
样本1: [5.1cm, 3.5cm, 1.4cm, 0.2cm] → Setosa
样本2: [7.0cm, 3.2cm, 4.7cm, 1.4cm] → Versicolor
样本3: [6.3cm, 3.3cm, 6.0cm, 2.5cm] → Virginica
```

## 🏥 示例2：乳腺癌诊断（二分类）

### 数据来源
- **来源**：sklearn内置数据集（威斯康星乳腺癌数据集）
- **加载方式**：`from sklearn.datasets import load_breast_cancer`

### 数据信息
| 属性 | 值 |
|------|-----|
| **样本总数** | 569个患者 |
| **特征数** | 30个（医学特征） |
| **训练集** | 455样本 |
| **测试集** | 114样本 |

### 特征详解（30个医学特征）

这些特征来自**显微镜图像分析**，对每个肿瘤细胞核计算以下指标：

#### 三大类指标，每类10个特征（3×10=30）

**1. 基础指标（10个）**
```
- mean radius          → 平均半径
- mean texture         → 平均纹理
- mean perimeter       → 平均周长
- mean area            → 平均面积
- mean smoothness      → 平均光滑度
- mean compactness     → 平均紧凑度
- mean concavity       → 平均凹陷度
- mean concave points  → 平均凹陷点数
- mean symmetry        → 平均对称度
- mean fractal dimension → 平均分形维数
```

**2. 标准差指标（10个）**
```
std radius, std texture, std perimeter, ... (同上，但是标准差)
```

**3. 最大值指标（10个）**
```
max radius, max texture, max perimeter, ... (同上，但是最大值)
```

### 分类标签
```
类别0: 恶性肿瘤 (Malignant)  → 212个
类别1: 良性肿瘤 (Benign)     → 357个
```

### 任务
**预测**：给定肿瘤的30个医学特征 → 判断是恶性还是良性
```
输入: [半径, 纹理, 周长, 面积, ...]  (30维向量)
输出: 肿瘤类型 (恶性 or 良性)
```

### 为什么有30个特征？
因为医学上需要从**多个角度**（半径、纹理等）、用**多个统计量**（均值、标准差、最大值）来描述肿瘤特征，才能准确判断。

## 🎯 示例3：多分类（三个随机类别）

### 数据来源
- **来源**：sklearn生成的**人造数据**（`make_blobs`）
- **特征**：完全随机，**没有现实意义**

### 数据信息
| 属性 | 值 |
|------|-----|
| **样本总数** | 500个点 |
| **特征数** | 2个（只有2维，易于可视化） |
| **类别数** | 3个（类别0, 1, 2） |
| **训练集** | 400样本 |
| **测试集** | 100样本 |

### 特征
```
特征1: X1  → 随机数值（0-10之间）
特征2: X2  → 随机数值（0-10之间）
```

### 类别
```
类别0: 第一个随机类群 → ~167个
类别1: 第二个随机类群 → ~167个
类别2: 第三个随机类群 → ~166个
```

### 任务
**预测**：给定两个数值 → 判断属于哪个类别
```
输入: [X1, X2]
输出: 类别 (0, 1, 或 2)
```

### ⚠️ 特点
- **无现实意义**：只是用来演示多分类算法
- **易于可视化**：2维数据可以直接画出来看决策边界
- **用途**：学习 One-vs-Rest 多分类策略

## ⚖️ 示例4：阈值调整（不平衡数据）

### 数据来源
- **来源**：sklearn生成的**人造数据**（`make_classification`）
- **特点**：类别**不平衡**（90%负类，10%正类）

### 数据信息
| 属性 | 值 |
|------|-----|
| **样本总数** | 1000 |
| **特征数** | 20 |
| **正类比例** | 10%（少数类） |
| **负类比例** | 90%（多数类） |

### 任务
**演示**：不同的**决策阈值**如何影响 准确率、精确率、召回率、F1分数

```
阈值0.3: 预测更多正类
阈值0.5: 平衡（默认）
阈值0.7: 预测更少正类
阈值0.9: 很难预测为正类
```

### 为什么需要调整阈值？
在医学诊断中，可能更在乎**不漏诊**(召回率)，而不是准确率。
调整阈值可以权衡精确率和召回率。

## 🛡️ 示例5：正则化对比

### 数据来源
- **来源**：sklearn生成的**人造数据**（`make_classification`）
- **特点**：高维数据（50个特征，但只有10个有用）

### 数据信息
| 属性 | 值 |
|------|-----|
| **样本总数** | 200 |
| **总特征数** | 50个 |
| **有用特征** | 10个（n_informative=10） |
| **冗余特征** | 40个（噪音） |

### 任务
**演示**：不同的正则化方式如何处理冗余特征

```
无正则化:  使用所有50个特征（容易过拟合）
L2正则化:  惩罚大系数，但保留所有特征
L1正则化:  把不重要特征的系数变为0（特征选择）
```

## 💡 核心概念

**特征（Features）** = 输入变量 = 描述样本的属性
```
Iris:      花的长宽高 → 4维向量
乳腺癌:    肿瘤特征 → 30维向量
示例3:     随机数值 → 2维向量
```

**标签（Label）** = 输出变量 = 要预测的结果
```
Iris:      品种 (Setosa/Versicolor/Virginica)
乳腺癌:    诊断 (恶性/良性)
```

**样本（Sample）** = 一组(特征, 标签)
```
一朵Iris花 = (4个测量值, 品种)
一个患者 = (30个医学指标, 诊断结果)
```
