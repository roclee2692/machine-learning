# 07 - CNN 与 RNN：两种经典深度学习架构

## 一句话总结

| 架构 | 核心思想 | 擅长什么 |
|------|---------|---------|
| **CNN**（卷积神经网络） | 用滑动窗口提取**局部空间特征** | 图像、空间数据 |
| **RNN**（循环神经网络） | 用隐藏状态记住**历史序列信息** | 文本、时间序列 |

---

## 0. 数据集：Fashion-MNIST

### 0.1 基本信息

| 属性 | 值 |
|------|-----|
| **发布者** | Zalando Research（2017） |
| **任务** | 10 类图像分类 |
| **图像尺寸** | 28×28 像素，灰度图（单通道） |
| **训练集** | 60,000 张 |
| **测试集** | 10,000 张 |
| **总大小** | ~30 MB（自动下载） |

### 0.2 为什么用 Fashion-MNIST 而不是 MNIST？

MNIST（手写数字 0-9）太简单了，随便一个模型都能跑到 99%+，根本分不出差距。

Fashion-MNIST 是 MNIST 的"升级版替代品"：
- **格式完全相同**：28×28 灰度图，10 类，60k+10k——任何跑 MNIST 的代码可以直接换上
- **难度更高**：衣服/鞋比手写数字复杂得多，不同模型的差距更明显
- **更有现实意义**：衣服分类是真实的电商场景

### 0.3 10 个类别

| 编号 | 类别 | 英文 | 视觉特征 |
|------|------|------|---------|
| 0 | T恤/上衣 | T-shirt/top | 有袖子、有领口 |
| 1 | 裤子 | Trouser | 长条形、两条裤腿 |
| 2 | 套衫 | Pullover | 类似T恤但更厚重 |
| 3 | 裙子 | Dress | 上窄下宽、无裤腿 |

### 0.5 对比实验中的意义

Fashion-MNIST 在这个实验里起到了**放大镜**的作用——让三种架构的差异一目了然：

| 模型 | 怎么"看"这张图 | 效果 |
|------|--------------|------|
| **RNN** | 按行从上往下"读"，但读到下半身时已经忘了上半身长什么样 | ~81% |
| **LSTM** | 同样按行读，但有选择地记住"领口是V领"、"有两条裤腿"等关键特征 | ~86% |
| **CNN** | 用卷积核扫描全图，直接提取"领口形状"、"裤腿轮廓"等空间纹理 | ~90% |

**关键洞察**：衣服的分类关键在于**空间结构**（领口、袖子、裤腿），而不是行与行之间的序列关系。这就是为什么 CNN 在图像任务上天然优于 RNN/LSTM。

## 1. CNN — 卷积神经网络

### 1.1 直觉理解

想象你看一张照片，你不是一次性看完所有像素，而是：
- 先用一个小窗口（比如 3×3）扫描局部区域，提取"边缘"、"纹理"等低级特征
- 然后逐步合并，从"边缘"→"眼睛"→"脸"这样的层次结构

**关键操作：**
```
输入图像 → [卷积核扫描] → 特征图 → [池化压缩] → 更小的特征图 → ... → 全连接 → 分类
```

### 1.2 核心组件

| 组件 | 作用 | 类比 |
|------|------|------|
| **卷积层 (Conv)** | 用小滤波器提取局部特征 | 用放大镜逐块看图 |
| **池化层 (Pool)** | 压缩特征图，保留重要信息 | 摘要：把 4 个像素合成 1 个 |
| **激活函数 (ReLU)** | 引入非线性 | 决定哪些特征"激活" |
| **全连接层 (FC)** | 最终分类 | 综合所有特征做决策 |

### 1.3 为什么 CNN 适合图像？

1. **局部连接**：每个卷积核只看局部区域，符合图像的局部相关性
2. **权重共享**：同一个卷核在整个图像上滑动，大幅减少参数
3. **平移不变性**：猫在左上角还是右下角，CNN 都能识别

### 1.4 一个典型 CNN 的参数量计算

```
Conv2d(1, 16, 3x3):  参数 = 16 × (1 × 3 × 3 + 1) = 160
Conv2d(16, 32, 3x3): 参数 = 32 × (16 × 3 × 3 + 1) = 4,640
Linear(1568, 128):   参数 = 1568 × 128 + 128 = 200,832
Linear(128, 10):     参数 = 128 × 10 + 10 = 1,290
总计: ~206,922
```

## 2. RNN — 循环神经网络

### 2.1 直觉理解

想象你读一句话："今天天气___"，要预测下一个词。
- 你不是只看最后一个字，而是按顺序读："今"→"天"→"天"→"气"
- 每读一个字，你的"记忆"（隐藏状态）就更新一次
- 最终用这个记忆来预测

**关键公式：**
```
h_t = tanh(W_hh · h_{t-1} + W_xh · x_t + b)
```

### 2.2 LSTM — 长短期记忆网络

普通 RNN 有**梯度消失**问题：序列一长，早期信息就"忘光了"。

LSTM 引入了三个"门"来控制信息流：

| 门 | 作用 | 类比 |
|----|------|------|
| **遗忘门 (Forget)** | 决定丢弃哪些旧信息 | "这个不重要，忘掉" |
| **输入门 (Input)** | 决定写入哪些新信息 | "这个很重要，记下来" |
| **输出门 (Output)** | 决定输出哪些信息 | "根据记忆，回答这个问题" |

```
细胞状态 C_t = 遗忘门 × C_{t-1} + 输入门 × 候选值
隐藏状态 h_t = 输出门 × tanh(C_t)
```

### 2.3 为什么 RNN 适合序列数据？

1. **时序建模**：天然处理变长序列，记住"前面发生了什么"
2. **参数共享**：同一组权重在每个时间步复用
3. **灵活输入输出**：可以处理各种序列→序列的映射

## 3. CNN vs RNN 对比

### 3.1 结构对比

```
CNN: 输入 → [Conv → Pool] × N → Flatten → FC → 输出
     特点：并行处理，一次看全部空间

RNN: x₁ → h₁ → x₂ → h₂ → ... → xₙ → hₙ → FC → 输出
     特点：串行处理，逐步读入序列
```

### 3.2 性能对比（Fashion-MNIST）

| 指标 | CNN | LSTM |
|------|-----|------|
| 测试准确率 | ~90-92% | ~85-88% |
| 训练速度 | 快（并行计算） | 慢（串行计算） |
| 参数效率 | 高（权重共享） | 中等 |
| 适合的数据 | 有空间结构的（图像） | 有时序结构的（文本/序列） |

### 3.3 在 Fashion-MNIST 上的对比逻辑

- **CNN 的做法**：用 3×3 卷积核扫描图像，提取"边缘"、"纹理"、"形状"等空间特征
- **LSTM 的做法**：把 28×28 的图像当成 28 个时间步，每步读入一行（28 维），试图从行序列中学习模式
- **为什么 CNN 更好**：衣服/鞋的分类关键在于**空间纹理**（领口形状、鞋底纹路），而不是行与行之间的序列关系

## 4. 什么时候用哪个？

| 场景 | 推荐 | 原因 |
|------|------|------|
| 图像分类 | CNN | 空间特征提取 |
| 目标检测 | CNN | 需要定位空间位置 |
| 文本分类 | 1D CNN 或 LSTM | 都可以，CNN 更快 |
| 机器翻译 | LSTM / Transformer | 需要长距离依赖 |
| 语音识别 | LSTM / Transformer | 时序信号 |
| 时间序列预测 | LSTM | 时序依赖 |
| 视频理解 | CNN + LSTM | 空间 + 时序 |

## 5. 历史小知识

- **1989**：LeCun 提出第一个成功的 CNN（LeNet），用于手写数字识别
- **1997**：Hochreiter & Schmidhuber 发明 LSTM
- **2012**：AlexNet 在 ImageNet 上大胜，CNN 成为计算机视觉主流
- **2014**：Yoon Kim 提出 TextCNN，CNN 开始用于 NLP
- **2017**：Transformer 论文发表，逐渐取代 RNN 在 NLP 中的地位
- **现在**：CNN 仍是 CV 主力，NLP 已全面转向 Transformer

## 6. 关键术语速查

| 术语 | 含义 |
|------|------|
| **卷积核 (Kernel/Filter)** | 小矩阵，用于提取局部特征 |
| **步长 (Stride)** | 卷积核每次移动的距离 |
| **填充 (Padding)** | 在输入边缘补零，保持输出尺寸 |
| **感受野 (Receptive Field)** | 一个输出神经元"看到"的输入区域大小 |
| **时间步 (Time Step)** | RNN 中序列的每个位置 |
| **隐藏状态 (Hidden State)** | RNN 在每个时间步的记忆 |
| **细胞状态 (Cell State)** | LSTM 的长期记忆载体 |
| **门控机制 (Gating)** | LSTM 用 sigmoid 控制信息流的方式 |
